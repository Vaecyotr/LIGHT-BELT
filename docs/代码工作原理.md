# 代码工作原理 — 从输入到输出的完整数据流

> 本文严格基于现有源码，描述一条完整的数据流：视频/音频输入 → 分析 → 灯效 → 输出。

---

## 数据流总览

```
输入视频 + 音频
  → 媒体读取 (media/)
  → 视频特征分析 (analysis/video.py)
  → 音频特征分析 (analysis/audio.py)
  → EffectContext 上下文
  → 灯效计算 (effects/)
  → PixelFrame 灯光帧
  → 多区域映射 (mapping/)
  → 输出发送 (outputs/)
```

---

## 步骤1: 媒体读取

**对应文件**: `light_engine/media/__init__.py`
**类**: `VideoReader`, `AudioReader`

### VideoReader
- **输入**: 视频文件路径 (MP4等)
- **做什么**: 用OpenCV打开视频，读取帧
- **输出**: BGR格式的numpy数组 (uint8), 如 `(height, width, 3)`
- **关键方法**: `read_frame(timestamp)` — 可按时间戳跳转读取
- **为什么需要**: 原始视频文件是压缩的，需要解码才能分析颜色

### AudioReader
- **输入**: 音频文件路径 (WAV/FLAC等)
- **做什么**: 用soundfile读取音频为浮点数组
- **输出**: 1维float32数组, 如 `[0.1, -0.2, 0.3, ...]`
- **关键方法**: `get_window_at(start_time, window_size)` — 按时间窗口切片
- **为什么需要**: 音频需要按窗口分段分析，不能一次分析整个文件

## 步骤2: 视频特征分析

**对应文件**: `light_engine/analysis/video.py`
**类**: `VideoAnalyzer`

- **输入**: BGR numpy数组 (来自VideoReader或合成数据)
- **做什么**:
  1. 缩放到160×90 (降低计算量)
  2. 过滤黑边像素(边缘5% + 阈值<15)
  3. 计算平均色 (排除黑边后的所有像素均值)
  4. 计算主色 (8×8×8=512色量化直方图，取最多的颜色)
  5. 分区颜色 (3×3网格: left/center/right/top/bottom)
  6. 计算亮度 (灰度均值/255)
  7. 计算饱和度 (HSV S通道均值/255)
  8. 场景变化检测 (相邻帧差异)
  9. 时间平滑 (EMA指数平滑)
- **输出**: `VideoFeatures` 对象
  ```python
  VideoFeatures(
      timestamp=1.0,
      average_rgb=(0.2, 0.5, 0.2),    # 平均RGB
      dominant_rgb=(0.3, 0.6, 0.1),   # 主色RGB
      zone_colors={                     # 分区颜色
          "left": (0.1, 0.3, 0.1),
          "right": (0.3, 0.7, 0.2),
          "center": (0.2, 0.5, 0.2),
          "top": (0.15, 0.4, 0.15),
          "bottom": (0.1, 0.2, 0.1),
      },
      brightness=0.5,                  # 亮度 [0,1]
      saturation=0.6,                  # 饱和度 [0,1]
      scene_change=0.0,                # 场景变化 [0,1]
  )
  ```
- **下一步由谁接收**: Engine → EffectContext → 灯效

## 步骤3: 音频特征分析

**对应文件**: `light_engine/analysis/audio.py`
**类**: `AudioAnalyzer`

- **输入**: 1维float32音频采样数组 (来自AudioReader或合成数据)
- **做什么**:
  1. 去除直流偏置 (减去均值)
  2. 计算RMS能量 (sqrt(mean(samples²)))
  3. FFT频谱分析 (Hanning窗, 缓存窗口函数)
  4. 低/中/高频段能量 (bass:20-200Hz, mid:200-2000Hz, treble:2000-12000Hz)
  5. 动态归一化 (P95滚动历史, 3秒窗口)
  6. Spectral flux (频谱变化量→瞬态检测)
  7. 节拍检测 (flux超过阈值, 200ms冷却)
  8. 静音判断 (RMS<0.01)
- **输出**: `AudioFeatures` 对象
  ```python
  AudioFeatures(
      timestamp=1.0,
      rms=0.614,        # 总体能量 [0,1]
      bass=0.814,       # 低频能量 [0,1]
      mid=0.150,        # 中频能量 [0,1]
      treble=0.050,     # 高频能量 [0,1]
      spectral_flux=0.3, # 频谱变化 [0,1]
      beat=True,         # 是否检测到节拍
      onset=0.3,         # onset强度 [0,1]
      silence=False,     # 是否静音
  )
  ```
- **下一步由谁接收**: Engine → EffectContext → 灯效

---

## 步骤4: 构建EffectContext

**对应文件**: `light_engine/models.py`
**类**: `EffectContext`

- **输入**: VideoFeatures + AudioFeatures + 配置参数
- **做什么**: 把所有信息打包成一个上下文对象
- **输出**: `EffectContext` 对象
  ```python
  EffectContext(
      timestamp=1.0,              # 当前时间
      delta_time=0.033,           # 距上帧时间
      video_features=video_f,     # 视频特征(None=无视频)
      audio_features=audio_f,     # 音频特征(None=无音频)
      global_brightness=0.85,     # 全局亮度上限
      speed=1.0,                  # 速度倍率
      intensity=1.0,              # 强度倍率
      mode_parameters={           # 灯效参数
          "strip_defs": [...],    # 灯带定义列表
          "zone_defs": [...],     # 区域定义列表
      },
  )
  ```
- **下一步由谁接收**: 灯效的 `process()` 方法

## 步骤5: 灯效计算

**对应文件**: `light_engine/effects/` (12个文件)
**类**: 每种灯效一个类

以 **VIDEO_AUDIO_FUSION** 为例 (`light_engine/effects/video_audio_fusion.py`):

- **输入**: EffectContext
- **做什么**:
  1. 从video_features取基础色相和区域颜色 (权重0.65)
  2. 从audio_features.rms计算目标亮度 (权重0.35)
  3. 用AttackReleaseEnvelope平滑亮度变化
  4. 低频(bass)计算脉冲扩散幅度
  5. 中频(mid)调整饱和度
  6. 高频(treble)产生每像素微闪(上限0.4)
  7. 节拍触发短促亮度提升
  8. 静音时保持环境光(不熄灭)
- **输出**: `PixelFrame` 对象
  ```python
  PixelFrame(
      timestamp=1.0,
      strips=[
          DigitalStrip(strip_id="ceiling_left", pixel_count=144,
              pixels=[(0.12,0.09,0.14), (0.16,0.12,0.18), ...]),
          DigitalStrip(strip_id="ceiling_right", pixel_count=144,
              pixels=[(0.12,0.09,0.14), ...]),
          ...  # 共6条
      ],
      zones=[
          ZoneOutput(zone_id="ceiling_left",
              color=RGBWColor(r=0.09, g=0.23, b=0.09, w=0.0, brightness=0.48)),
          ...  # 共6个
      ],
      metadata={"video_rgb": (0.2,0.5,0.2), "audio_rms":0.614, "beat":True},
  )
  ```

数值示例说明:
- 视频显示绿色画面 → 所有RGB中G最高
- 音频低音强 (bass=0.814) → beat=True
- 融合后亮度=0.48 (适中)
- 第一条灯带144像素，每个像素都是近似暗绿色

## 步骤6: 区域映射

**对应文件**: `light_engine/mapping/__init__.py`
**类**: `Layout`

- **输入**: 配置文件 `config/layout.yaml`
- **做什么**: 定义哪些灯带/区域存在，以及它们映射到视频的哪个区域
- **关键映射**:
  - `ceiling_left`, `ceiling_right` → 视频top区域
  - `wall_left` → 视频left区域
  - `wall_right` → 视频right区域
  - `front`, `rear` → 视频center区域
- **为什么需要**: 视频分析得出各区域颜色后，需要知道每个物理灯带应该用哪个区域的颜色

---

## 步骤7: 输出

**对应文件**: `light_engine/outputs/`

### SimulatorOutput (`outputs/simulator_output.py`)
- 把PixelFrame缓冲在内存队列中
- 终端模拟器读取并显示为彩色方块

### JsonOutput (`outputs/json_output.py`)
- 把PixelFrame序列化为JSON，写入文件
- 每行一个JSON对象 (JSON Lines格式)
- 示例输出见 `新手使用说明.md` 第十五节

### NullOutput (`outputs/null_output.py`)
- 丢弃所有数据, 用于性能测试

### UdpOutput (`outputs/udp_output.py`)
- 把数字灯带的像素编码为二进制UDP包
- 预留ESP32-S3接收 (当前仅编码, 未实际发送)

### SerialOutput (`outputs/serial_output.py`)
- 把RGBW区域编码为11字节串口帧
- 预留STM32接收 (当前仅编码和内存transport)

## 无视频/无音频时的降级路径

| 情况 | 行为 |
|------|------|
| 仅合成数据 | 内置生成器模拟视频颜色循环和音频鼓点 |
| 仅视频 | 视频分析正常运行, 音频特征=None |
| 仅音频 | 音频分析正常运行, 视频特征=None |
| 两者都无 | 自动切换到CALM或STATIC模式 |
| 无GUI环境 | 终端模拟器或JSONL导出 |

## 配置加载流程

**对应文件**: `light_engine/config/__init__.py`
**类**: `Config`

1. 程序启动时调用 `Config.get_instance()`
2. 加载4个YAML文件: system.yaml, layout.yaml, effects.yaml, outputs.yaml
3. 深合并为一个字典
4. 通过 `config.get("system.output_fps")` 点号路径取值
5. 不在每帧重新加载 (只在启动时加载一次)
6. 不执行任意代码 (使用 `yaml.safe_load`)
