# 灯光算法原型 — 新手使用说明

> **写给谁看**：没有Python编程经验的人。不需要提前安装任何软件。
> **前提条件**：你有一台Windows电脑，会双击打开文件。

---

## 一、这个项目是做什么的

这是一个**灯光效果控制系统**。

它能够：
- 分析视频画面的颜色（比如电影、MV画面）
- 分析音乐的节奏和能量（比如歌曲的低音鼓点）
- 根据画面颜色和音乐节奏，**自动控制LED灯带**的颜色和亮度

最终目标是把这套系统安装在**加压供氧舱**里，让舱内的灯光跟随视频和音乐变化，创造沉浸式体验。

---

## 二、当前能看到什么效果

**不需要任何硬件**（不需要LED灯带、不需要单片机），你现在就可以在电脑屏幕上看到：

1. **6条虚拟灯带**在屏幕上逐像素显示颜色
2. **6个灯光区域**的RGBW（红绿蓝白）颜色
3. 灯光效果随时间变化（颜色流动、呼吸、跑马灯等）
4. 当前帧率（FPS）
5. 视频分析结果（主色、亮度）
6. 音频分析结果（低音能量、中音能量、高音能量、是否检测到节拍）

---

## 三、当前还不能做什么

- 不能连接真实LED灯带（需要STM32、ESP32硬件）
- 不能连接真实串口或WiFi设备
- 没有漂亮的图形界面（目前是黑底彩色文字的终端窗口）
- 没有在RK3588 ARM芯片上实测（目前在普通Windows电脑上运行）

## 七、如何切换灯效

目前有12种灯效模式。在命令行中可以用不同的 `--effect` 参数切换，但目前最简便的方式是**修改配置文件**：

1. 用记事本打开 `config\effects.yaml`
2. 找到 `active: "demo"` 这一行
3. 把 `"demo"` 改成以下任意一个名字：
   - `"static"` — 固定颜色
   - `"breath"` — 呼吸灯（缓慢明暗变化）
   - `"color_wave"` — 颜色波浪（颜色沿灯带流动）
   - `"chase"` — 跑马灯
   - `"comet"` — 流星效果
   - `"audio_pulse"` — 跟随音乐能量呼吸
   - `"bass_pulse"` — 跟随低音脉冲
   - `"spectrum"` — 不同频段映射到不同区域
   - `"video_ambient"` — 跟随视频画面颜色
   - `"video_audio_fusion"` — 视频颜色+音乐能量融合
   - `"calm"` — 安静模式（缓慢变化）
   - `"demo"` — 自动轮流切换
4. 保存文件
5. 双击 `01_启动演示.bat` 重新运行

## 八、如何改变运行时间

在命令行中可以用 `--duration` 参数：

```
.python\python.exe -m light_engine demo --duration 60
```

这会运行60秒。改为你想要的时间即可。时间单位是**秒**。

或者在Windows资源管理器中：右键点击 `.bat` 文件 → 编辑 → 修改 `--duration 30` 中的数字 → 保存 → 双击运行。

---

## 九、如何改变亮度

修改配置文件 `config\system.yaml`：

1. 用记事本打开 `config\system.yaml`
2. 找到 `max_brightness: 0.85`
3. 改成你想要的亮度（0~1之间，1=100%）
   - `0.3` = 30%亮度
   - `0.5` = 50%亮度
   - `0.85` = 85%亮度（默认）
4. 保存文件
5. 重新运行

---

## 十、如何改变灯效速度

修改配置文件 `config\effects.yaml` 中对应效果的 `speed` 参数。

例如跑马灯速度：
1. 用记事本打开 `config\effects.yaml`
2. 找到 `chase:` 下面的 `speed: 2.0`
3. 改成你想要的倍速：
   - `1.0` = 正常速度
   - `0.5` = 半速
   - `3.0` = 三倍速
4. 保存文件
5. 重新运行

## 十一、如何选择视频文件

如果你有视频文件（MP4格式），可以这样运行：

1. 把视频文件放到项目文件夹里（或记住它的完整路径）
2. 打开命令行（Win+R → 输入 `cmd` → 回车）
3. 进入项目目录：
   ```
   cd /d A:\BaiduNetdiskDownload\LIGHT-BELT
   ```
4. 运行命令：
   ```
   .python\python.exe -m light_engine run --video "你的视频文件路径.mp4" --effect video_ambient
   ```

> **路径中有空格怎么办**：用英文双引号把路径包起来。例如：
> ```
> --video "D:\我的视频\电影.mp4"
> ```

## 十五、JSON导出文件是什么

`output\exported_lights.jsonl` 是一个**纯文本文件**，可以用记事本打开。

每一行是一帧灯光数据，格式如下：

```json
{
  "timestamp": 0.0,
  "strips": [
    {
      "strip_id": "ceiling_left",
      "pixel_count": 144,
      "pixels": [[26,18,26], [37,29,37], ...]
    }
  ],
  "zones": [
    {
      "zone_id": "ceiling_left",
      "r": 23, "g": 58, "b": 23, "w": 0,
      "brightness": 0.475
    }
  ],
  "metadata": {
    "video_rgb": [0.2, 0.5, 0.2],
    "audio_rms": 0.614,
    "bass": 0.814,
    "beat": true
  }
}
```

字段含义：
- `timestamp`: 这一帧的时间（秒）
- `strips`: 每条数字灯带，每颗像素的RGB值（0-255）
- `zones`: 每个RGBW区域的红绿蓝白值和亮度
- `metadata`: 分析结果（视频颜色、音频能量、检测到的节拍）

## 十六、测试通过是什么意思

双击 `03_运行测试.bat` 后，你会看到大量绿色 `PASSED`。

`100 passed` 表示所有162个自动测试全部通过。这意味着：
- 颜色转换正确（RGB转HSV转RGBW不会出错）
- 音频分析正确（低频声音检测为低频、高频声音检测为高频）
- 视频分析正确（红色画面检测到红色）
- 协议编码正确（发给STM32的11个字节格式正确）
- 所有灯效都能正常生成输出
- 没有任何计算错误（没有NaN、没有无穷大、没有除零）

## 十九、常见错误及解决方法

### 错误1：双击.bat文件后窗口一闪就没了

**原因**：`.python` 文件夹可能被移动或删除。

**解决**：确保 `.bat` 文件和 `.python` 文件夹在同一个目录下。

### 错误2：提示找不到视频文件

**原因**：视频文件路径不正确或文件不存在。

**解决**：
- 检查文件路径拼写
- 路径中有空格时加双引号
- 把视频文件复制到项目文件夹里，直接写文件名

### 错误3：提示 `No module named 'light_engine'`

**原因**：`.python\python311._pth` 文件中的路径配置不正确。

**解决**：确保 `.python\python311._pth` 文件中包含 `..` 这一行。

### 错误4：模拟器窗口显示乱码

**原因**：终端不支持ANSI颜色编码。

**解决**：在较新的Windows Terminal或CMD中运行；或使用导出模式查看数据。
